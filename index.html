Relational Algebra

A formal, mathematical set of operations on relations. Input is one or more tables; the output is always a new table. It is the theory that SQL is built on.

OperatorSymbolDoes whatSQL equivalent
SelectionσPicks rows matching a conditionWHERE
ProjectionπPicks columns (removes duplicates)SELECT DISTINCT
RenameρRenames a relation / attributeAS
Cartesian product×Every row of A with every row of BFROM A, B
Join⋈Cartesian product + a match conditionJOIN … ON
Union∪Rows in A or B (no duplicates)UNION
Intersection∩Rows in A and BINTERSECT / IN
Set difference−Rows in A but not BEXCEPT / NOT IN

The two basics in pictures

Examples 
σ marks>80 (Student)                         -- rows where marks exceed 80
π Fname, Ssn (EMPLOYEE)                       -- only these two columns
ρ EMPNAME ( π Fname (EMPLOYEE) )              -- relabel the result as EMPNAME
π Name ( Student ⋈ Student.Did=Department.Dnumber Department )  -- join then project
Translate the words "has no / does not" → set difference −  ·  "has both X and Y" → intersection ∩. And remember A − B ≠ B − A (order matters), while ∪ and ∩ are commutative.
02

Insert · Delete · Update

The three commands that change data. (SELECT, which only reads, is next.) DML = Data Manipulation Language — don't confuse it with DDL's CREATE / ALTER / DROP.

INSERT — add rows

-- Full row: values in the SAME ORDER as CREATE TABLE
INSERT INTO EMPLOYEE
VALUES ('Richard','K','Marini','653298653','1952-12-30',
        '98 Oak Forest, Katy, TX','M',37000,'987654321',4);

-- Partial row: name the columns; the rest become NULL
INSERT INTO EMPLOYEE (Fname, Lname, Ssn)
VALUES ('Richard','Marini','653298653');

-- Many rows at once
INSERT INTO EMPLOYEE (Fname, Lname, Ssn)
VALUES ('Richard','Marini','653298653'),
       ('Ali','Ahmed','123456889');
Trap Skip the column list and you must supply every value in exact order; any omitted column must allow NULL or the insert fails.

DELETE — remove rows

DELETE FROM EMPLOYEE WHERE Lname = 'Brown';   -- conditional
DELETE FROM EMPLOYEE;                          -- DANGER: removes EVERY row
Trap No WHERE = the whole table is emptied. Deletes touch one table at a time unless a foreign key uses ON DELETE CASCADE.

UPDATE — modify existing rows

UPDATE PROJECT
SET Plocation = 'Bellaire', Dnum = 5
WHERE Pnumber = 10;

UPDATE EMPLOYEE
SET Salary = Salary * 1.1     -- new value can depend on the old value (10% raise)
WHERE Dno = 5;
Trap Forgetting WHERE in UPDATE changes every row in the table.
03

The SELECT Statement

The most important statement in the course. Its shape: choose columns, name tables, filter rows.

SELECT        -- which columns
FROM              -- which tables
WHERE   ;          -- which rows

      
Core building blocks

      
SELECT Fname, Lname, Address FROM EMPLOYEE;           -- specific columns
SELECT * FROM EMPLOYEE;                                -- everything
SELECT DISTINCT Salary FROM EMPLOYEE;                  -- remove duplicate rows
SELECT * FROM EMPLOYEE WHERE Dno IS NULL;              -- NULL test
SELECT Fname FROM EMPLOYEE ORDER BY Fname DESC;        -- sort (default ASC)
SELECT * FROM EMPLOYEE WHERE Salary BETWEEN 30000 AND 50000;  -- inclusive range
SELECT Fname, Lname, 1.1*Salary FROM EMPLOYEE WHERE Dno=5;    -- arithmetic

      
Trap Never write = NULL. NULL is "unknown", so the test is never true. Always IS NULL / IS NOT NULL.


      
LIKE — pattern matching (a guaranteed exam topic)

      
      
WHERE Address LIKE '%Houston,TX%'   -- contains the substring anywhere
WHERE Fname   LIKE 'K%'             -- starts with K
WHERE Lname   LIKE '%d'             -- ends with d
WHERE Fname   LIKE 'A%A'            -- starts AND ends with A
WHERE Pname   LIKE '_r%'           -- 'r' in the 2nd position
WHERE Pname   NOT LIKE '%H%'        -- does not contain H
WHERE Salary  LIKE '_____'          -- exactly 5 characters (5-digit salary)


      
LIMIT — the "Nth highest" trick (MySQL)

      
SELECT Fname FROM EMPLOYEE ORDER BY Salary DESC LIMIT 0,1;  -- highest paid
SELECT Fname FROM EMPLOYEE ORDER BY Salary DESC LIMIT 1,1;  -- 2nd highest

      
LIMIT offset, count → skip offset rows, then return count rows.

    

    
    

      
04
Joins

      
A join stitches tables together side by side on a related column — it adds columns. Master the four flavours and you can answer almost any retrieval question.


      
Cross product — the starting point

      
Every row of A paired with every row of B, with no condition. This is rarely what you want, but it's the basis of all joins.

      

      
The four join types — as Venn diagrams

      

        

          
          
INNER JOIN
Only rows that match in both tables.

        

        

          
          
LEFT JOIN
All of the left table + matches; unmatched right = NULL.

        

        

          
          
RIGHT JOIN
All of the right table + matches; unmatched left = NULL.

        

        

          
          
FULL OUTER
Everything from both sides. Not in MySQL — fake it with LEFT∪RIGHT.

        

      


      
Inner join — three equivalent syntaxes

      
-- (a) implicit (comma + WHERE)
SELECT s.RollNumber, s.Name, d.Dname
FROM Student s, Department d
WHERE s.Did = d.DNumber;

-- (b) explicit — preferred
SELECT s.RollNumber, s.Name, d.Dname
FROM Student s INNER JOIN Department d ON s.Did = d.DNumber;

-- (c) with an extra filter
SELECT s.RollNumber, s.Name, d.Dname
FROM Student s INNER JOIN Department d ON s.Did = d.DNumber
WHERE d.Dname = 'FOIT';


      
Outer joins

      
SELECT s.Name, d.Dname FROM Student s LEFT  JOIN Department d ON s.Did=d.DNumber; -- keeps every student
SELECT s.Name, d.Dname FROM Student s RIGHT JOIN Department d ON s.Did=d.DNumber; -- keeps every dept

      
Decision rule The phrase "whether they have one or not" / "show all A" → an OUTER JOIN. The phrase "only those that have a matching…" → an INNER JOIN.


      
Natural join & USING

      
SELECT * FROM Student NATURAL JOIN Department;     -- auto-joins on same-named column, no ON
SELECT * FROM Student JOIN Department USING (Did);  -- name the shared column, shown once


      
Self join — a table joined to itself (two aliases)

      
SELECT e.Fname AS Employee, s.Fname AS Supervisor
FROM EMPLOYEE e JOIN EMPLOYEE s ON e.Super_ssn = s.Ssn;


      
Multi-table join (employee → project)

      
SELECT e.Fname, p.Pname
FROM EMPLOYEE e
JOIN WORKS_ON w ON e.Ssn = w.Essn
JOIN PROJECT  p ON w.Pno = p.Pnumber;

      
Several lecture slides had typos (IINER, Prjoect, Wroks_on). The corrected forms are above — write clean SQL in the exam.

    


    
    

      
05
Aggregates · GROUP BY · HAVING

      
An aggregate function takes a set of values and returns a single value — the bridge from "list every row" to "summarise per group".


      

        
FunctionReturnsKey detail

        
COUNTnumber of rows/valuesCOUNT(*) counts NULLs & dupes; COUNT(col) ignores NULLs; COUNT(DISTINCT col) ignores both

        
SUMtotalnumeric only

        
AVGaveragenumeric only

        
MAX / MINlargest / smallestworks on numbers, dates, text

      


      
SELECT SUM(Salary), MAX(Salary), MIN(Salary), AVG(Salary) FROM EMPLOYEE;
SELECT Dno, COUNT(*), AVG(Salary) FROM EMPLOYEE GROUP BY Dno;   -- per department

      
Golden rule Every column in SELECT that is not inside an aggregate must appear in GROUP BY.


      
WHERE vs HAVING

      
SELECT d.Dname, COUNT(dl.Dlocation) AS total_locations
FROM DEPARTMENT d JOIN DEPT_LOCATIONS dl USING (Dnumber)
GROUP BY d.Dname
HAVING COUNT(dl.Dlocation) > 2;     -- filter on the aggregated result

      
WHERE filters rows before grouping; HAVING filters groups after aggregating. HAVING does nothing without GROUP BY.

Nested Queries & Set Operations

A subquery is a query inside another query (usually in WHERE). The inner query runs first and hands a value or list up to the outer query.

IN / NOT IN — membership

-- IN is shorthand for many ORs
SELECT Name FROM Student WHERE Name IN ('Maria','Maida','Madiha');

-- Subquery feeding IN: employees in the Research department
SELECT * FROM EMPLOYEE
WHERE Dno IN (SELECT Dnumber FROM DEPARTMENT WHERE Dname='Research');

-- Multi-level: employees on a project located in Stafford
SELECT * FROM EMPLOYEE
WHERE Ssn IN (SELECT Essn FROM WORKS_ON
              WHERE Pno IN (SELECT Pnumber FROM PROJECT WHERE Plocation='Stafford'));

-- NOT IN: employees who are not managers
SELECT * FROM EMPLOYEE WHERE Ssn NOT IN (SELECT Mgr_ssn FROM DEPARTMENT);

Set operations, and how to fake them

OperationSQLAlternative
Union (no dupes)UNION— only way
Union (keep dupes)UNION ALL—
IntersectionINTERSECTIN subquery / a JOIN
DifferenceEXCEPTNOT IN subquery
SELECT Name FROM Student WHERE Address LIKE '%Lahore%'
UNION
SELECT Name FROM Teacher WHERE Address LIKE '%Lahore%'
ORDER BY Name DESC;          -- ORDER BY goes ONCE, at the very end
Union compatibility Both SELECTs need the same number of columns with matching domains.
JOIN Combines columns — tables side by side. The result has more columns.
UNION Stacks rows — one query below another. The result has more rows.

ANY vs ALL

SELECT Name FROM Employee WHERE Salary > ANY (SELECT Salary FROM Employee WHERE DeptID=10);
-- equivalent to:  Salary > (SELECT MIN(Salary) FROM Employee WHERE DeptID=10)

SELECT Name FROM Employee WHERE Salary > ALL (SELECT Salary FROM Employee WHERE DeptID=10);
-- equivalent to:  Salary > (SELECT MAX(Salary) FROM Employee WHERE DeptID=10)
07

Correlated Subqueries · EXISTS

A correlated subquery references a column from the outer query, so it re-runs once for every outer row. EXISTS is the classic correlated pattern.

EXISTS TRUE when the subquery returns at least one row → "has some / at least one".
NOT EXISTS TRUE when the subquery returns no rows → "none / never / does not".
-- Employees who work on at least one project (IN vs correlated EXISTS)
SELECT * FROM EMPLOYEE WHERE Ssn IN (SELECT DISTINCT Essn FROM WORKS_ON);
SELECT * FROM EMPLOYEE e WHERE EXISTS
       (SELECT * FROM WORKS_ON w WHERE w.Essn = e.Ssn);

-- Employees with NO dependents
SELECT * FROM EMPLOYEE e WHERE NOT EXISTS
       (SELECT * FROM DEPENDENT d WHERE d.Essn = e.Ssn);

-- Customers who placed at least one order
SELECT Name FROM Customer c WHERE EXISTS
       (SELECT * FROM Orders o WHERE c.ID = o.C_id);
Why prefer NOT EXISTS If a NOT IN subquery returns even one NULL, the whole result silently becomes empty. NOT EXISTS is NULL-safe — say this for marks.

The "break it into pieces" method

For a hard query, solve each sub-part, then assemble. Projects involving an employee named Smith — as worker or as manager:

SELECT DISTINCT Pname FROM PROJECT
WHERE Pnumber IN (SELECT w.Pno FROM WORKS_ON w JOIN EMPLOYEE e ON e.Ssn=w.Essn
                  WHERE e.Lname='Smith')                                  -- as worker
   OR Pnumber IN (SELECT p.Pnumber FROM PROJECT p
                  JOIN DEPARTMENT d ON p.Dnum=d.Dnumber
                  JOIN EMPLOYEE  e ON d.Mgr_ssn=e.Ssn
                  WHERE e.Lname='Smith');                                  -- as manager

ER → Relational Mapping

Turning an ER diagram into tables. The standard seven steps — most exam ER questions are one of these.

#ER elementBecomes
1Strong entityIts own table; pick the PK.
2Weak entityOwn table; PK = owner's PK + partial key; owner's PK as FK.
31:1 relationshipPut one side's PK as a FK in the other.
41:N relationshipPut the "1" side's PK as a FK on the "N" side. (e.g. Dno in EMPLOYEE)
5M:N relationshipNew junction table; PK = both entities' PKs + relationship attributes. (WORKS_ON)
6Multivalued attributeSeparate table (owner PK + the attribute), PK = both. (DEPT_LOCATIONS)
7N-ary relationshipOne table holding all participants' PKs as a composite key.
Spot the pattern A table with a composite key built from two foreign keys = an M:N junction. A table of (PK + one attribute) = a multivalued attribute (e.g. StudentHobby(St_ID, Hobby)).

Solved — SportsDB Nesting Set

All fifteen practice questions, fully answered. Tables: Team, Player, Tournament, MatchDetails, PlayerPerformance, Sponsor (assumptions noted where wording is ambiguous).

Players whose salary > the average salary of their team (correlated).
SELECT p.PlayerName, p.Salary FROM Player p
WHERE p.Salary > (SELECT AVG(p2.Salary) FROM Player p2 WHERE p2.TeamID = p.TeamID);
Highest-paid player from each team.
SELECT p.PlayerName, p.TeamID, p.Salary FROM Player p
WHERE p.Salary = (SELECT MAX(p2.Salary) FROM Player p2 WHERE p2.TeamID = p.TeamID);
Teams having at least one player older than 35.
SELECT TeamName FROM Team
WHERE TeamID IN (SELECT TeamID FROM Player WHERE Age > 35);
Tournaments whose prize money > the average prize money.
SELECT TournamentName, PrizeMoney FROM Tournament
WHERE PrizeMoney > (SELECT AVG(PrizeMoney) FROM Tournament);
Players who scored more runs than the average runs in their match (correlated).
SELECT pp.PlayerID, pp.MatchID, pp.RunsScored FROM PlayerPerformance pp
WHERE pp.RunsScored > (SELECT AVG(pp2.RunsScored)
                       FROM PlayerPerformance pp2 WHERE pp2.MatchID = pp.MatchID);
Teams that have won at least one match.
SELECT TeamName FROM Team
WHERE TeamID IN (SELECT WinnerTeamID FROM MatchDetails);
Players who never played any match (NULL-safe NOT EXISTS).
SELECT p.PlayerName FROM Player p
WHERE NOT EXISTS (SELECT * FROM PlayerPerformance pp WHERE pp.PlayerID = p.PlayerID);
Matches where no player scored more than 100 runs.
SELECT m.MatchID FROM MatchDetails m
WHERE NOT EXISTS (SELECT * FROM PlayerPerformance pp
                  WHERE pp.MatchID = m.MatchID AND pp.RunsScored > 100);
Players whose rating is higher than all teammates (assume avg rating per player).
SELECT p.PlayerName FROM Player p
WHERE (SELECT AVG(pp.Rating) FROM PlayerPerformance pp WHERE pp.PlayerID = p.PlayerID)
    > ALL (SELECT AVG(pp2.Rating)
           FROM Player p2 JOIN PlayerPerformance pp2 ON p2.PlayerID = pp2.PlayerID
           WHERE p2.TeamID = p.TeamID AND p2.PlayerID <> p.PlayerID
           GROUP BY p2.PlayerID);
Teams having more sponsors than team 'Lahore Lions'.
SELECT t.TeamName FROM Team t
WHERE (SELECT COUNT(*) FROM Sponsor s WHERE s.TeamID = t.TeamID)
    > (SELECT COUNT(*) FROM Sponsor s JOIN Team t2 ON s.TeamID = t2.TeamID
       WHERE t2.TeamName = 'Lahore Lions');
Players with a higher average rating than their team average.
SELECT p.PlayerName FROM Player p
WHERE (SELECT AVG(pp.Rating)  FROM PlayerPerformance pp WHERE pp.PlayerID = p.PlayerID)
    > (SELECT AVG(pp2.Rating)
       FROM PlayerPerformance pp2 JOIN Player p2 ON pp2.PlayerID = p2.PlayerID
       WHERE p2.TeamID = p.TeamID);
Matches where players from both teams scored centuries (≥100).
SELECT m.MatchID FROM MatchDetails m
WHERE EXISTS (SELECT * FROM PlayerPerformance pp JOIN Player p ON pp.PlayerID=p.PlayerID
              WHERE pp.MatchID=m.MatchID AND pp.RunsScored>=100 AND p.TeamID=m.Team1ID)
  AND EXISTS (SELECT * FROM PlayerPerformance pp JOIN Player p ON pp.PlayerID=p.PlayerID
              WHERE pp.MatchID=m.MatchID AND pp.RunsScored>=100 AND p.TeamID=m.Team2ID);
Teams that never lost any match (never appear as a non-winning participant).
SELECT TeamName FROM Team t
WHERE t.TeamID NOT IN (
    SELECT Team1ID FROM MatchDetails WHERE WinnerTeamID <> Team1ID
    UNION
    SELECT Team2ID FROM MatchDetails WHERE WinnerTeamID <> Team2ID);
Players whose total runs > their team's average total runs (derived table + correlation).
SELECT p.PlayerName FROM Player p
WHERE (SELECT SUM(pp.RunsScored) FROM PlayerPerformance pp WHERE pp.PlayerID = p.PlayerID)
    > (SELECT AVG(team_totals.player_total)
       FROM (SELECT SUM(pp2.RunsScored) AS player_total
             FROM PlayerPerformance pp2 JOIN Player p2 ON pp2.PlayerID = p2.PlayerID
             WHERE p2.TeamID = p.TeamID
             GROUP BY p2.PlayerID) AS team_totals);
Sponsors whose amount > all other sponsors of the same team.
SELECT s.SponsorName, s.SponsorshipAmount FROM Sponsor s
WHERE s.SponsorshipAmount > ALL (
    SELECT s2.SponsorshipAmount FROM Sponsor s2
    WHERE s2.TeamID = s.TeamID AND s2.SponsorID <> s.SponsorID);

Solved — Join Queries Sheet

The JOIN_QUERIES sheet, answered on the COMPANY schema.

Department managers + their department names; manager name starts with 'J'.
SELECT e.Fname, e.Lname, d.Dname
FROM EMPLOYEE e JOIN DEPARTMENT d ON e.Ssn = d.Mgr_ssn
WHERE e.Fname LIKE 'J%';
Details of departments located in Houston.
SELECT d.* FROM DEPARTMENT d
JOIN DEPT_LOCATIONS dl ON d.Dnumber = dl.Dnumber
WHERE dl.Dlocation = 'Houston';
Department names + names and salaries of the male employees.
SELECT d.Dname, e.Fname, e.Lname, e.Salary
FROM DEPARTMENT d JOIN EMPLOYEE e ON d.Dnumber = e.Dno
WHERE e.Sex = 'M';
Employees with the names of their supervisors (self join).
SELECT e.Fname AS Employee, s.Fname AS Supervisor
FROM EMPLOYEE e JOIN EMPLOYEE s ON e.Super_ssn = s.Ssn;
Female employees + projects on which they spent at least 10 hours.
SELECT e.Fname, e.Salary, p.Pname, w.Hours
FROM EMPLOYEE e
JOIN WORKS_ON w ON e.Ssn = w.Essn
JOIN PROJECT  p ON w.Pno = p.Pnumber
WHERE e.Sex = 'F' AND w.Hours >= 10;
Dependents whose guardian works on a project in Houston or Stafford.
SELECT DISTINCT dep.Dependent_name
FROM DEPENDENT dep
JOIN EMPLOYEE e ON dep.Essn = e.Ssn
JOIN WORKS_ON w ON e.Ssn = w.Essn
JOIN PROJECT  p ON w.Pno = p.Pnumber
WHERE p.Plocation IN ('Houston','Stafford');
Projects administered by a department whose manager is James Borg.
SELECT p.* FROM PROJECT p
JOIN DEPARTMENT d ON p.Dnum = d.Dnumber
JOIN EMPLOYEE   e ON d.Mgr_ssn = e.Ssn
WHERE e.Fname = 'James' AND e.Lname = 'Borg';

Last-Minute Cheat Sheet

Phrase → tool translation

The question says…Reach for
"all A, whether matched or not"OUTER JOIN
"only A that have a matching B"INNER JOIN / IN / EXISTS
"does not have / never / no"NOT IN / NOT EXISTS / −
"has both X and Y"two IN/EXISTS (AND) / INTERSECT
"greater than every / all"> ALL ≡ > MAX
"greater than some / any"> ANY ≡ > MIN
"for each / per group"GROUP BY
"groups where count/avg…"HAVING
"highest / Nth highest"ORDER BY … DESC LIMIT n-1,1
"contains / starts / ends with"LIKE with % and _
"no value / empty"IS NULL

Don't-lose-marks checklist

  • IS NULL, never = NULL.
  • UPDATE/DELETE with no WHERE hits every row.
  • Cross product rows = rows(A) × rows(B).
  • Aggregates can't go in WHERE → use HAVING.
  • Every non-aggregated SELECT column must be in GROUP BY.
  • COUNT(*) counts NULLs; COUNT(col) doesn't.
  • UNION needs equal column counts + matching domains.
  • Prefer NOT EXISTS over NOT IN when NULLs are possible.
  • Self-join needs two different aliases.
  • FULL OUTER JOIN isn't in MySQL — use LEFT UNION RIGHT.
