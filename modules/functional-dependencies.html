Functional Dependencies & Attribute Closure

A functional dependency X → Y says: whenever two rows agree on X, they must agree on Y — X determines Y. FDs are the raw material of schema design. From a set of FDs, Armstrong's axioms let us infer new ones, and the attribute-closure algorithm computes everything an attribute set determines — the key tool for finding keys and normalizing.

5Pitfalls & exam tips

⚠️ Common mistakes

  • An FD is a constraint on all legal states, not something you can "prove" from one sample table — a table can only violate it.
  • X → Y does not imply Y → X. Determination is one-directional.
  • Confusing the closure of attributes X⁺ (a set of attributes) with the closure of FDs F⁺ (a set of FDs).
  • Forgetting to re-scan all FDs after adding attributes — the closure loop repeats until nothing changes.

💡 Exam tips

  • The three Armstrong axioms (reflexivity, augmentation, transitivity) are sound and complete — everything else derives from them.
  • X → Y is implied by F ⟺ Y ⊆ X⁺. This single test answers most questions.
  • X is a superkey ⟺ X⁺ = all attributes of R.
  • Closure is the engine behind finding keys (next module) and checking normal forms.
