Nested Queries — IN / NOT IN and ANY / ALL

A nested (sub)query is a SELECT inside another query. A simple subquery runs once, producing a set of values that the outer query tests against. IN checks membership; ANY and ALL compare against the smallest or largest of that set. Watch the inner query resolve first, then feed the outer filter.

5Pitfalls & exam tips

⚠️ Common mistakes

  • NOT IN with NULLs is a classic trap: if the subquery returns even one NULL, NOT IN yields no rows (everything becomes UNKNOWN).
  • Using = with a subquery that returns multiple rows — that errors; use IN.
  • Confusing > ANY (greater than the minimum) with > ALL (greater than the maximum).

💡 Exam tips

  • A non-correlated subquery runs once; mentally replace it with its result set, then read the outer query.
  • = ANY ≡ IN; <> ALL ≡ NOT IN — handy rewrites.
  • Many IN subqueries can be rewritten as joins, often with better performance.
  • Prefer NOT EXISTS over NOT IN when NULLs are possible (next module).
