Set Operations — UNION, INTERSECT, MINUS

Set operators combine the result sets of two queries vertically (stacking rows), treating them as mathematical sets. UNION is everything in either; INTERSECT is what's in both; MINUS (a.k.a. EXCEPT) is what's in the first but not the second. They require union compatibility — matching column counts and types.

5Pitfalls & exam tips

⚠️ Common mistakes

  • Mismatched column counts or incompatible types → not union compatible, query rejected.
  • Forgetting UNION removes duplicates (and sorts) — use UNION ALL if you need every row and speed.
  • Assuming column names must match — only the count and types (by position) must; output uses the first query's names.
  • MINUS is Oracle's keyword; standard SQL / PostgreSQL use EXCEPT.

💡 Exam tips

  • Set operators stack results vertically; joins combine horizontally.
  • ORDER BY goes once, at the very end of the whole compound query.
  • UNION ALL avoids the dedup/sort cost — prefer it when duplicates can't occur.
  • A INTERSECT B ≡ A MINUS (A MINUS B) — a handy identity.
