Normalization — 1NF, 2NF & 3NF

Normalization restructures tables to remove redundancy and the update, insert, and delete anomalies it causes. Each normal form adds one rule: 1NF demands atomic values, 2NF bans partial dependencies on a composite key, and 3NF bans transitive dependencies through non-prime attributes. Each step decomposes a table into smaller, cleaner ones.

5Pitfalls & exam tips

⚠️ Common mistakes

  • Checking 2NF when the key is a single attribute — partial dependencies are impossible, so it's automatically 2NF.
  • Decomposing in a way that loses information — decompositions should be lossless (the join rebuilds the original).
  • Confusing 2NF (depends on part of the key) with 3NF (depends on a non-key attribute).
  • Forgetting to identify candidate keys first — every normal form is defined relative to them.

💡 Exam tips

  • Order of attack: find candidate keys → 1NF (atomic) → 2NF (remove partial) → 3NF (remove transitive).
  • 3NF test for X→A: it's OK if X is a superkey or A is a prime attribute.
  • Anomalies are the "why": redundancy causes update, insertion, and deletion anomalies.
  • A good decomposition is both lossless-join and ideally dependency-preserving.
